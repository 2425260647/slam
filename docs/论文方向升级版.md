# 基于 16 线激光雷达的三维点云到二维占据栅格地图生成优化与 AMCL 鲁棒定位研究

## 研究主线
本研究不再只关注原始二维地图的后处理，而是将问题上移到地图生成阶段：
1. 从 16 线激光雷达三维点云出发，优化 3D 点云到 2D 占据栅格地图的投影过程；
2. 在此基础上进行结构感知的地图增强与质量评估；
3. 以 AMCL 定位效果作为最终验证目标。

## 为什么这样改
- 更接近地图源头，创新层级更高；
- 保留你现有的 `pointcloud_to_grid` 和 `my_navigation` 工程基础；
- 仍然能做出 `M0/M1/M2/M3` 实验闭环；
- 更容易把“地图质量提升”与“定位提升”建立因果链。

## 三个创新点
### 创新点一：面向定位任务的 3D 到 2D 投影优化
结合雷达高度、地面/墙体分层、局部结构先验，改进从 `PointCloud2` 到 `OccupancyGrid` 的投影规则，减少虚假障碍和断墙。

### 创新点二：地图质量量化指标体系
建立墙体连续度、自由区连通度、噪点密度、边界清晰度、未知区比例五个指标，用于不同地图版本的横向比较。

### 创新点三：地图质量与 AMCL 鲁棒定位的对应关系
通过 `M0 -> M1 -> M2 -> M3` 的消融实验，量化地图质量变化对定位 RMSE、收敛速度和稳定性的影响。

## 实验路线
1. 录制 `rosbag`：`/velodyne_points`、`/tf`、`/odom`
2. 生成基线地图 `M0`
3. 输出优化版地图 `M1/M2/M3`
4. 用 AMCL 在相同参考点做定位对比
5. 汇总地图质量指标和定位误差

## 结论
这条路线比单纯二维后处理更强，因为它把研究重点从“修图”提升到了“地图生成质量控制 + 定位验证”。
